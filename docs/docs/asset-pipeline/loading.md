# Loading Assets

Assets are loaded at runtime from a binary content folder resolved via a JSON manifest.

## AssetLoader

Static class for loading models, textures, and shaders.

### Loading Models
```csharp
// Load by relative path (extension isnt required)
Model model = AssetLoader.LoadModel("characters/robot");

// Cast to AnimatedModel if needed
AnimatedModel animatedModel = (AnimatedModel)AssetLoader.LoadModel("characters/robot");
```
### Loading Textures
```csharp
GLTexture texture = AssetLoader.LoadTexture("textures/brick-albedo");
```
### Loading Shaders 
Handled by [Shaders Helpers](../rendering/shaders.md) autogenerated code.

## Caching

All loaded assets are internally cached in static dictionaries:
```csharp
_loadedModels:   Dictionary<string, Model>
_loadedTextures: Dictionary<string, GLTexture>
_loadedShaders:  Dictionary<string, GLShader>
```
Subsequent calls with the same name return the cached instance:
```csharp
var model1 = AssetLoader.LoadModel("characters/robot");
var model2 = AssetLoader.LoadModel("characters/robot");
// model1 == model2  (same instance)
```


## Asset Manifest

A JSON file that maps asset names to file paths.

### Example

```json
{
    "Assets": [
        {
            "RelativePath": "Models/characters/robot.bin",
            "Type": 0
        },
        {
            "RelativePath": "Textures/brick-albedo",
            "Type": 1
        },
        {
            "RelativePath": "Models/characters/brick-albedo-extracted",
            "Type": 2
        },
        {
          "RelativePath": "Shaders/animated/animated.frag",
          "Type": 3
        },
        {
          "RelativePath": "Shaders/animated/animated.vert",
          "Type": 3
        }
    ],
    "Namespace": "Phoenix.Framework.ShaderHelpers",
    "DarkTheme": true
}
```

### AssetType Enum

| Value | Name | Description |
|-------|------|-------------|
| 0 | `Model` | Binary model file (`.bin`) |
| 1 | `Texture` | Binary texture file (`.bin`) |
| 3 | `ExtTexture` | Binary texture, extracted from a model (`.bin`) |
| 3 | `Shader` | Shader stage file (`.vert` / `.frag`) |
| 4 | `Unknown` | Unrecognized asset type |

## Error Handling
Shaders are loaded directly as text files, but compiled during Asset building to verify any errors.  

If an asset is not found in the manifest, `LoadModel()`, `LoadTexture()`, and `LoadShader()` will throw an exception. If a shader has no matching `.vert`/`.frag` pair, it throws with a descriptive error message.
